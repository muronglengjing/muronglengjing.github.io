运算符重载

重载规则

原有的语义不变

  • 不改变运算符的优先级
  • 不改变运算符的结合性
  • 不改变运算符需要的操作数

运算符函数

  • 类型 类名::operator op(参数表){
    //相对于该类的操作
    }

默认重载版本

  • =

    • 对象数据成员的复制

  • &

    • 返回对象的地址

用成员/友元函数重载运算符

区别

  • 有this指针
  • 无this指针
  • 无法重载= () [] ->

一元运算符

  • Object.operator op()
  • operator op(Object)

二元运算符

  • ObjectL.operator op(ObjectR)
  • operator op(ObjectL,ObjectR)

典型的重载

重载++、–

（参数int是一个伪参数，用于前置后置的区别。
函数体内不应该使用伪参数，不然会引起调用的二义性）

  • 前置

    • A &A::operator++()
    • friend A & operator++(A &)

  • 后置

    • A &A::operator++(int)
    • friend A & operator++(A &, int)

重载赋值运算符

  • 类型 & 类名::operator=(类名)

重载[]和()

  • 重载[]

    • 调用形式

      • 对象 [表达式]

  • 重载()

    • 调用形式

      • 对象(表达式表)
      • 表达式表可以为空

重载插入提取流

  • 必须重载为友元函数

  • 插入流

    • isteam &operator>>(isteam & input, A & a){
      //操作
      return input;
      }

  • 提取流

    • osteam &operator<<(osteam & ouput, A & a){
      //操作
      return output;
      }

类类型转换

默认转换

  • 可以通过构造函数来实现默认的类型转换

类类型转换

  • operator Type();

    • 函数声明

  • ClassA::operator Type(){
    return ClassA;
    }
